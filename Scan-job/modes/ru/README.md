# career-ops — Русские режимы (`modes/ru/`)

Этот каталог содержит русскоязычные версии режимов career-ops для соискателей, работающих на российском рынке труда или с русскоязычными вакансиями.

## Когда использовать?

Используй `modes/ru/`, если выполняется хотя бы одно из условий:

- Ты откликаешься на **русскоязычные вакансии** (hh.ru, Хабр Карьера, trudvsem.ru, rabota.ru, Superjob, Geekjob, budu.jobs)
- Твоё **резюме на русском** или ты переключаешься между RU и EN в зависимости от вакансии
- Тебе нужны ответы и сопроводительные письма на **естественном техническом русском**, а не машинный перевод
- Ты работаешь с **особенностями российского рынка**: ТК РФ, НДФЛ 13%, ДМС, испытательный срок, оформление по ТК / ГПХ / самозанятость, зарплата gross vs net

Если большинство вакансий на английском, используй стандартные режимы из `modes/`. Английские режимы работают и с русскоязычными вакансиями, но не учитывают специфику российского рынка в полном объёме.

## Как активировать?

### Способ 1 — Per session, по команде

Скажи Claude в начале сессии:

> "Используй русские режимы из `modes/ru/`."

или

> "Оценки и отклики на русском — используй `modes/ru/_shared.md` и `modes/ru/oferta.md`."

Claude будет читать файлы из этого каталога. Если режима нет в `modes/ru/` (например, `scan`, `batch`, `pdf`), используется оригинал из `modes/`.

### Способ 2 — Постоянно, через профиль

Добавь в `config/profile.yml`:

```yaml
language:
  primary: ru
  modes_dir: modes/ru
```

Напомни Claude в первой сессии: "Смотри в `profile.yml`, я указал `language.modes_dir`".

> Примечание: `language.modes_dir` — конвенция проекта (аналогично `modes/de/` и `modes/fr/`), а не автоматический механизм в коде. Claude следует ей как указанию при наличии напоминания. Если агент не подхватил настройку — напомните ещё раз.

## Что переведено?

Первая итерация охватывает режимы с наибольшим влиянием:

| Файл | Оригинал | Назначение |
|------|----------|------------|
| `_shared.md` | `modes/_shared.md` (EN) | Общий контекст, архетипы, глобальные правила, специфика рынка РФ |
| `oferta.md` | `modes/oferta.md` (ES) | Полная оценка вакансии (блоки A-F) |
| `apply.md` | `modes/apply.md` (ES) | Ассистент заполнения форм откликов |
| `pipeline.md` | `modes/pipeline.md` (ES) | Очередь URL-ов / Second Brain для собранных вакансий |
| `interview-prep.md` | `modes/interview-prep.md` (EN) | Подготовка к собеседованию |

Остальные режимы (`scan`, `batch`, `pdf`, `tracker`, `auto-pipeline`, `deep`, `contacto`, `ofertas`, `project`, `training`) остаются на EN/ES. Их содержимое — в основном tooling, пути и команды — не зависит от языка.

## Что остаётся на английском?

Сознательно не переведено, т.к. это стандартный tech-словарь:

- `cv.md`, `pipeline`, `tracker`, `report`, `score`, `archetype`, `proof point`
- Имена инструментов (`Playwright`, `WebSearch`, `WebFetch`, `Read`, `Write`, `Edit`, `Bash`)
- Статусы в трекере (`Evaluated`, `Applied`, `Interview`, `Offer`, `Rejected`)
- Код, пути, команды

Режимы используют технический русский, как его говорят в реальных инженерных командах: русский текст, английские термины там, где они общеприняты. Никакой принудительной русификации "pipeline" → "конвейер", "cv.md" → "резюме-файл".

## Вклад

Если хочешь улучшить перевод или добавить новый режим:

1. Открой issue с предложением (см. `CONTRIBUTING.md`)
2. Придерживайся словаря выше для единообразия
3. Переводи по смыслу и идиоматично — без дословных переводов
4. Сохраняй структурные элементы (блоки A-F, таблицы, код, инструкции для инструментов)
5. Протестируй с реальной русскоязычной вакансией (например, с hh.ru или Хабр Карьера)
